# 项目规则(与 .cursor/rules 同步)

> 本文件内容与 `.cursor/rules/*.mdc` 保持同步;修改任一侧规则时,请同步更新另一侧。

## 规则一:会话上下文落盘(AGENTS.md)

> 对应 `.cursor/rules/agent-context-to-agent-md.mdc`,每轮对话均适用。

在**每一轮**完成对用户的可见回复之前或刚结束之后(同一轮内最后一步执行),更新仓库根目录的 `AGENTS.md`:

### 必须写入的内容

- **日期与时间**(用户环境为准,如 2026-05-14)。
- **本轮用户诉求**:1~3 句概括用户想要什么。
- **本轮结论**:已做什么、关键决策(技术选型、接口约定等)。
- **涉及文件**:本轮创建/修改的路径列表(相对仓库根目录)。
- **未完成 / 风险**:若有阻塞、待确认项,简要列出。
- **建议下一步**:1~5 条可执行项。

### 格式与长度

- 在 `AGENTS.md` 维护 **「最新会话」** 区块(用二级标题 `## 最新会话`),**每轮用三级标题 `###` + 时间戳** 在区块顶部追加一条记录;**全部保留在 `AGENTS.md`,不做归档拆分**。
- 使用简体中文;避免粘贴密钥、Token、真实密码。
- 若本轮**未产生任何实质变更**且仅为闲聊/单句确认,可只写一行摘要,不必展开文件列表。

### 例外

- 若用户明确要求「不要写入 AGENTS.md」本轮可跳过,并在 `AGENTS.md` 最新会话中注明「用户要求跳过落盘」。
- 若本轮**仅为启动/停止前后端、运行 dev 服务**等日常运维操作,**且无代码或配置变更**,不写 `AGENTS.md`。

## 规则二:代码变更后的自检清单

> 对应 `.cursor/rules/post-code-change-verification.mdc`,在编写或修改 `**/*.{py,vue,ts,tsx,js}` 文件后适用。

在完成对仓库内 **Python / Vue / TypeScript / JavaScript** 文件的编写或修改后、结束本轮前,必须执行:

### 1. 完整性

- 重读**已编辑文件**的相邻上下文:导入是否齐全、导出/路由是否注册、配置项是否同步(如环境变量示例、Vite 代理)。
- 新增 API 时:请求/响应类型与后端字段是否一致;删除代码时:是否遗留死引用或未使用文件。

### 2. 准确性

- 逻辑分支是否覆盖错误路径(空数据、401/403、网络失败);异步与错误处理是否合理。
- 安全相关:鉴权、权限、敏感信息不落日志、不明文存密码。

### 3. 自动化检查(按项目可选执行)

- 对**改动路径**检查诊断/静态错误;若有报错,**优先修复**再交付。
- 若本轮改动了 `frontend/`:在项目根执行 `cd frontend && npm run build`(或至少 `npm run build`),失败则修复至通过。
- 若本轮改动了 `backend/`:执行 `cd backend && source .venv/bin/activate 2>/dev/null; python -c "from app.main import app"` 或等价导入检查;若有测试则运行相关用例。

### 4. 收尾

- 若引入新依赖:更新 `package.json` / `requirements.txt` 并在回复中一句说明。
- 若自检发现与需求不符,修正代码或明确告知用户缺口,**不把已知错误留到「以后再修」而不说明**。
