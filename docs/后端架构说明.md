## 后端架构说明（FastAPI + SQLAlchemy + Pydantic）

本文用于帮助快速理解本项目后端的目录结构与代码分层，适合对照代码阅读。

### 总览：一次请求的典型链路

- **入口**：`app/main.py` 创建 FastAPI 应用，注册中间件与各模块路由（统一 `/api/v1` 前缀），并挂载静态文件 `/static`。
- **依赖注入**：通过 `Depends(...)` 注入数据库 Session（`app/database.py:get_db`）、当前用户/管理员（`app/deps.py`）。
- **路由层（HTTP 层）**：`app/routers/*.py` 定义 URL、参数来源、鉴权依赖与返回结构；编排调用 repository/service。
- **数据访问层（Repository）**：`app/repositories/*.py` 负责数据库查询与持久化（SQLAlchemy select/join/分页/保存）。
- **业务层（Service）**：`app/services/*.py` 放业务规则、跨模块流程、序列化、支付/超时、AI embedding/检索等可复用能力。
- **数据结构**：
  - **ORM 模型**：`app/models/*.py`（对应数据库表）
  - **Schema**：`app/schemas/*.py`（对外 API 的请求/响应结构，Pydantic）
- **迁移**：`backend/alembic/` 维护数据库结构迁移脚本。

## 目录与文件职责

### `backend/app/main.py`（应用入口与组装）

你可以把它看成“把整个后端拼起来”的文件：

- 注册 CORS 中间件
- `include_router(...)` 把各模块路由挂到应用上，并统一加 `/api/v1` 前缀
- `app.mount("/static", ...)` 把 `uploads/` 目录作为静态资源目录暴露出去（用于图片访问）
- 提供 `/health` 健康检查与 `/api/v1` 简单接口索引

示例：统一注册路由前缀（节选）

```python
app.include_router(orders.router, prefix="/api/v1")
app.include_router(admin.router, prefix="/api/v1")
```

最终路径 = `"/api/v1"` + 模块自身 `router.prefix` + endpoint 路径。

---

### `backend/app/config.py`（配置与环境变量）

集中读取环境变量并提供 `settings` 给各处使用，例如：

- `database_url`
- `debug`
- `cors_origins`
- 各种 AI/种子数据开关（例如启动时是否 seed、是否批量建 embedding）

---

### `backend/app/database.py`（数据库连接与 Session）

提供 SQLAlchemy Engine 与 Session 工厂，并暴露 FastAPI 可用的依赖函数：

- `SessionLocal`: 生成 `Session`
- `get_db()`: `yield` 一个 Session，请求结束后自动 close

它的职责是“提供数据库会话”，不关心业务。

---

### `backend/app/deps.py`（依赖注入：鉴权与权限）

统一放置 `Depends(...)` 依赖函数，典型包括：

- `get_current_user`: 解析 JWT，找到用户并检查禁用
- `get_current_admin`: 基于 `get_current_user` 再校验 `role == admin`
- `get_optional_user`: 可选登录态（未登录返回 None）

路由里通过 `Depends(get_current_user)` / `Depends(get_current_admin)` 来声明权限要求。

## 分层：Router / Repository / Service / Schema / Model

### 1）`app/routers/*` 路由层（HTTP 层）

路由层负责：

- 定义 URL 与 HTTP 方法：`@router.get(...)` / `@router.post(...)`
- 声明参数来源：path/query/body，以及 `Depends(...)` 注入的对象（db/user/admin）
- 声明返回结构：`response_model=...`
- **编排调用**：把参数整理后调用 repositories/services，然后返回 schemas

例如 `app/routers/orders.py` 的一个列表接口（节选）：

```python
@router.get("", response_model=list[OrderListItem])
def list_buyer_orders(
    status_filter: Optional[str] = None,
    db: Session = Depends(get_db),
    user: User = Depends(get_current_user),
) -> list[OrderListItem]:
    rows = OrderRepository(db).list_by_buyer(user.id, status_filter)
    return [serialize_order_list_item(o, user) for o in rows]
```

可以看到 router 并不自己写 SQL，而是：

- 通过 `Depends(get_db)` 获得 `db`
- 通过 `Depends(get_current_user)` 获得 `user`
- 调用 `OrderRepository(db).list_by_buyer(...)` 拿订单
- 调用 `serialize_order_list_item(...)` 组装返回结构

> 小项目里，router 里也可能包含部分“业务编排”（例如退款通过时既要改订单状态又要恢复库存又要写审计），后续可视复杂度逐步下沉到 service。

---

### 2）`app/repositories/*` Repository（数据访问层）

Repository 负责“数据库怎么查/怎么存”，常见内容：

- SQLAlchemy `select(...)` / `where(...)` / `order_by(...)` / `limit(...)`
- `selectinload/join` 预加载关联数据
- `save/create/delete`（add/commit/refresh）

例：`app/repositories/order.py`（节选）

- `_ORDER_LOAD` 决定“订单详情需要预加载哪些关联”（买家、卖家、商品及图片）
- `get_detail/list_all/list_by_buyer/list_by_seller` 封装查询细节

Repository 的目标是让路由/服务不需要关心“SQL 细节”。

---

### 3）`app/services/*` Service（业务层）

Service 放更“可复用”的业务能力，典型包括：

- **序列化**：把 ORM 组合成对外 Schema（例如生成 `cover_image`、计算 `payment_expires_at`）
- **状态机规则/流程**：订单超时取消、支付抽象、退款流程等
- **跨模块能力**：AI embedding、RAG 检索、评分聚合、审计日志记录

例：`app/services/order_serializers.py`（节选）

- 将 `Order` ORM 组装成 `OrderDetail` / `OrderListItem`
- 计算支付超时时间 `payment_expires_at`
- 从商品图片中推导 `cover_image`

这类逻辑放在 service，便于在不同路由复用、便于测试与演进。

---

### 4）`app/schemas/*`（Pydantic：请求/响应结构）

Schema 定义“接口对外长什么样”，用于：

- **请求体**（body）校验：例如退款驳回原因、重置密码、创建订单等
- **响应体**（response_model）约束：例如订单详情、商品详情、管理员订单列表项等

它与 ORM 模型不同：

- ORM 是“数据库里怎么存”
- Schema 是“API 返回/接收什么字段”

---

### 5）`app/models/*`（SQLAlchemy ORM：数据库表结构）

每个 model 对应一张表或一组关联表，例如：

- `user.py` 用户
- `product.py` 商品
- `order.py` 订单
- `product_favorite.py` 收藏关系
- `admin_audit_log.py` 管理员操作审计

模型变更后通常需要配合 Alembic 迁移更新数据库结构。

## 模块划分与对应文件（读代码时的索引）

- **认证与用户**：`routers/auth.py`、`routers/users.py`、`deps.py`、`core/security.py`
- **商品与收藏**：`routers/products.py`、`models/product*.py`、`repositories/product.py`、`services/product_serializers.py`
- **订单与支付/退款**：`routers/orders.py`、`repositories/order.py`、`services/order_payment_timeout.py`、`services/payment.py`
- **管理后台**：`routers/admin.py`（统一加 `dependencies=[Depends(get_current_admin)]` 约束管理员权限）
- **AI**：`routers/ai.py`、`services/ai/*`、`repositories/ai.py`、`models/ai_*`

## 常见困惑与判断口诀

- **“这个逻辑该放哪？”**
  - **查数据库/写数据库** → Repository
  - **业务规则/跨模块流程/计算字段/序列化** → Service
  - **URL/鉴权依赖/参数声明/把调用串起来** → Router
  - **对外字段结构** → Schema
  - **表结构与关系** → Model
